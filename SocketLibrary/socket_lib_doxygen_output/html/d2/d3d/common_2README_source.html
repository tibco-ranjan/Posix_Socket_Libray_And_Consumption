2 Calls which sets errno.
3 ****************************
4 To diagnose errors from system calls and library functions,
5 we use errMsg(), errExit(), err_exit(), and errExitEN().
6 
7 =================================================================================
8 void errMsg(const char *format, ...);
9 void errExit(const char *format, ...);
10 void err_exit(const char *format, ...);
11 void errExitEN(int errnum, const char *format, ...);
12 ================================================================================
13 
14 The errMsg() function prints a message on standard error. Its argument list is the
15 same as for printf(), except that a terminating newline character is automatically
16 appended to the output string. The errMsg() function prints the error text corresponding
17 to the current value of errno—this consists of the error name, such as
18 EPERM, plus the error description as returned by strerror()—followed by the formatted
19 output specified in the argument list.
20 
21 The errExit() function operates like errMsg(), but also terminates the program, by calling exit().
22 
23 The err_exit() function is similar to errExit(), but differs in two respects:
24 􀁺 It doesn’t flush standard output before printing the error message.
25 􀁺 It terminates the process by calling _exit() instead of exit().
26 This causes the process to terminate without flushing stdio buffers or invoking exit handlers.
27 
28 The errExitEN() function is the same as errExit(), except that instead of printing
29 the error text corresponding to the current value of errno, it prints the text corresponding
30 to the error number (thus, the EN suffix) given in the argument errnum.
31 
32 Mainly, we use errExitEN() in programs that employ the POSIX threads API.
33 Unlike traditional UNIX system calls, which return –1 on error, the POSIX threads
34 functions diagnose an error by returning an error number (i.e., a positive number
35 of the type normally placed in errno) as their function result. (The POSIX threads
36 functions return 0 on success.)
37 
38 Example:
39  s = pthread_create(&thread, NULL, func, &arg);
40  if (s != 0)
41  errExitEN(s, "pthread_create");

46 To diagnose other types of errors, we use fatal(), usageErr(), and cmdLineErr().

52 
53 The fatal() function is used to diagnose general errors, including errors from
54 library functions that don’t set errno. Its argument list is the same as for printf(),
55 except that a terminating newline character is automatically appended to the output
56 string. It prints the formatted output on standard error and then terminates
57 the program as with errExit().
58 
59 
60 The usageErr() function is used to diagnose errors in command*line argument
61 usage. It takes an argument list in the style of printf() and prints the string Usage:
62 followed by the formatted output on standard error, and then terminates the program
63 by calling exit().
64 
65 
66 The cmdLineErr() function is similar to usageErr(), but is intended for diagnosing
67 errors in the command*line arguments specified to a program.
